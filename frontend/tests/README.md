# 前端测试目录

## 📁 目录说明

本目录包含历史数据统计分析工具的前端UI自动化测试脚本。

## 📋 测试文件列表

### 🎯 主要测试脚本

- **`nlp-frontend-ui-test.js`** - NLP前端UI自动化测试主脚本
  - 测试NLP功能页面的UI交互效果
  - 包含词频分析、时间轴分析、地理分析等功能测试
  - 生成详细的测试报告和截图

- **`nlp-ui-test.js`** - NLP UI测试辅助脚本
- **`ui-test-complete.js`** - 完整UI测试套件
- **`ui-test.js`** - 基础UI测试脚本
- **`run-ui-test.js`** - 测试运行器
- **`screenshot.js`** - 截图工具脚本
- **`test-puppeteer.js`** - Puppeteer测试工具
- **`simple-test.js`** - 简单测试脚本

## 🚀 运行测试

### 前置条件

1. 确保Node.js环境已安装
2. 安装依赖包：
   ```bash
   npm install puppeteer
   ```
3. 确保前端应用正在运行（通常在 http://localhost:3000）

### 运行主要测试

```bash
# 进入测试目录
cd frontend/tests

# 运行NLP前端UI测试
node nlp-frontend-ui-test.js

# 运行其他测试
node ui-test-complete.js
```

## 📝 维护说明

1. **添加新测试**：在对应的测试脚本中添加新的测试方法
2. **更新测试数据**：修改测试脚本中的测试数据
3. **调整测试配置**：修改浏览器启动参数和测试超时设置
4. **查看测试结果**：检查生成的报告和截图文件

## 🐛 故障排除

### 常见问题

1. **Chrome浏览器路径错误**
   - 检查Chrome安装路径
   - 更新脚本中的executablePath配置

2. **测试超时**
   - 检查前端应用是否正常运行
   - 增加测试超时时间

3. **元素定位失败**
   - 检查页面元素选择器
   - 确认页面加载完成

4. **权限问题**
   - 确保有写入测试报告和截图的权限
   - 检查目录权限设置
